     <h3 id="introduction">
      Introduction
     </h3>
     <p>
      Using
      <strong>
       for loops
      </strong>
      and
      <strong>
       <a href="https://www.digitalocean.com/community/tutorials/how-to-construct-while-loops-in-python-3">
        while loops
       </a>
      </strong>
      in Python allow you to automate and repeat tasks in an efficient manner.
     </p>
     <p>
      But sometimes, an external factor may influence the way your program runs. When this occurs, you may want your program to exit a loop completely, skip part of a loop before continuing, or ignore that external factor. You can do these actions with
      <code>
       break
      </code>
      ,
      <code>
       continue
      </code>
      , and
      <code>
       pass
      </code>
      statements.
     </p>
     <h2 id="break-statement">
      Break Statement
     </h2>
     <p>
      In Python, the
      <code>
       break
      </code>
      statement provides you with the opportunity to exit out of a loop when an external condition is triggered. You’ll put the
      <code>
       break
      </code>
      statement within the block of code under your loop statement, usually after a conditional
      <code>
       if
      </code>
      statement.
     </p>
     <p>
      Let’s look at an example that uses the
      <code>
       break
      </code>
      statement in a
      <code>
       for
      </code>
      loop:
     </p>
     <pre class="code-pre"><code class="code-highlight language-python">number = 0

for number in range(10):
    if number == 5:
        break    # break here

    print('Number is ' + str(number))

print('Out of loop')

</code></pre>
     <p>
      In this small program, the variable
      <code>
       number
      </code>
      is initialized at 0. Then a
      <code>
       for
      </code>
      statement constructs the loop as long as the variable
      <code>
       number
      </code>
      is less than 10.
     </p>
     <p>
      Within the
      <code>
       for
      </code>
      loop, there is an
      <code>
       if
      </code>
      statement that presents the condition that
      <em>
       if
      </em>
      the variable
      <code>
       number
      </code>
      is equivalent to the integer 5,
      <em>
       then
      </em>
      the loop will break.
     </p>
     <p>
      Within the loop is also a
      <code>
       print()
      </code>
      statement that will execute with each iteration of the
      <code>
       for
      </code>
      loop until the loop breaks, since it is after the
      <code>
       break
      </code>
      statement.
     </p>
     <p>
      To know when we are out of the loop, we have included a final
      <code>
       print()
      </code>
      statement outside of the
      <code>
       for
      </code>
      loop.
     </p>
     <p>
      When we run this code, our output will be the following:
     </p>
     <pre class="code-pre"><code><div class="secondary-code-label" title="Output">Output</div>Number is 0
Number is 1
Number is 2
Number is 3
Number is 4
Out of loop
</code></pre>
     <p>
      This shows that once the integer
      <code>
       number
      </code>
      is evaluated as equivalent to 5, the loop breaks, as the program is told to do so with the
      <code>
       break
      </code>
      statement.
     </p>
     <p>
      The
      <code>
       break
      </code>
      statement causes a program to break out of a loop.
     </p>
     <h2 id="continue-statement">
      Continue Statement
     </h2>
     <p>
      The
      <code>
       continue
      </code>
      statement gives you the option to skip over the part of a loop where an external condition is triggered, but to go on to complete the rest of the loop. That is, the current iteration of the loop will be disrupted, but the program will return to the top of the loop.
     </p>
     <p>
      The
      <code>
       continue
      </code>
      statement will be within the block of code under the loop statement, usually after a conditional
      <code>
       if
      </code>
      statement.
     </p>
     <p>
      Using the same
      <code>
       for
      </code>
      loop program as in the
      <a href="https://www.digitalocean.com/community/tutorials/how-to-use-break-continue-and-pass-statements-when-working-with-loops-in-python-3#break-statement">
       Break Statement
      </a>
      section above, we’ll use a
      <code>
       continue
      </code>
      statement rather than a
      <code>
       break
      </code>
      statement:
     </p>
     <pre class="code-pre"><code class="code-highlight language-python">number = 0

for number in range(10):
    if number == 5:
        continue    # continue here

    print('Number is ' + str(number))

print('Out of loop')

</code></pre>
     <p>
      The difference in using the
      <code>
       continue
      </code>
      statement rather than a
      <code>
       break
      </code>
      statement is that our code will continue despite the disruption when the variable
      <code>
       number
      </code>
      is evaluated as equivalent to 5. Let’s look at our output:
     </p>
     <pre class="code-pre"><code><div class="secondary-code-label" title="Output">Output</div>Number is 0
Number is 1
Number is 2
Number is 3
Number is 4
Number is 6
Number is 7
Number is 8
Number is 9
Out of loop
</code></pre>
     <p>
      Here,
      <code>
       Number is 5
      </code>
      never occurs in the output, but the loop continues after that point to print lines for the numbers 6-10 before leaving the loop.
     </p>
     <p>
      You can use the
      <code>
       continue
      </code>
      statement to avoid deeply nested conditional code, or to optimize a loop by eliminating frequently occurring cases that you would like to reject.
     </p>
     <p>
      The
      <code>
       continue
      </code>
      statement causes a program to skip certain factors that come up within a loop, but then continue through the rest of the loop.
     </p>
     <h2 id="pass-statement">
      Pass Statement
     </h2>
     <p>
      When an external condition is triggered, the
      <code>
       pass
      </code>
      statement allows you to handle the condition without the loop being impacted in any way; all of the code will continue to be read unless a
      <code>
       break
      </code>
      or other statement occurs.
     </p>
     <p>
      As with the other statements, the
      <code>
       pass
      </code>
      statement will be within the block of code under the loop statement, typically after a conditional
      <code>
       if
      </code>
      statement.
     </p>
     <p>
      Using the same code block as above, let’s replace the
      <code>
       break
      </code>
      or
      <code>
       continue
      </code>
      statement with a
      <code>
       pass
      </code>
      statement:
     </p>
     <pre class="code-pre"><code class="code-highlight language-python">number = 0

for number in range(10):
    if number == 5:
        pass    # pass here

    print('Number is ' + str(number))

print('Out of loop')

</code></pre>
     <p>
      The
      <code>
       pass
      </code>
      statement occurring after the
      <code>
       if
      </code>
      conditional statement is telling the program to continue to run the loop and ignore the fact that the variable
      <code>
       number
      </code>
      evaluates as equivalent to 5 during one of its iterations.
     </p>
     <p>
      We’ll run the program and consider the output:
     </p>
     <pre class="code-pre"><code><div class="secondary-code-label" title="Output">Output</div>Number is 0
Number is 1
Number is 2
Number is 3
Number is 4
Number is 5
Number is 6
Number is 7
Number is 8
Number is 9
Out of loop
</code></pre>
     <p>
      By using the
      <code>
       pass
      </code>
      statement in this program, we notice that the program runs exactly as it would if there were no conditional statement in the program. The
      <code>
       pass
      </code>
      statement tells the program to disregard that condition and continue to run the program as usual.
     </p>
     <p>
      The
      <code>
       pass
      </code>
      statement can create minimal classes, or act as a placeholder when working on new code and thinking on an algorithmic level before hammering out details.
     </p>
     <h2 id="conclusion">
      Conclusion
     </h2>
     <p>
      The
      <code>
       break
      </code>
      ,
      <code>
       continue
      </code>
      , and
      <code>
       pass
      </code>
      statements in Python will allow you to use
      <code>
       for
      </code>
      loops and
      <code>
       while
      </code>
      loops more effectively in your code.
     </p>
     <p>
      To work more with
      <code>
       break
      </code>
      and
      <code>
       pass
      </code>
      statements, you can follow our project tutorial “
      <a href="https://www.digitalocean.com/community/tutorials/how-to-create-a-twitterbot-with-python-3-and-the-tweepy-library">
       How To Create a Twitterbot with Python 3 and the Tweepy Library
      </a>
      .”
     </p>
