<p>Python's <code>time</code> module has a handy function called <code>sleep()</code>. Essentially, as the name implies, it pauses your Python program. <code>time.sleep()</code> is the equivalent to the Bash shell's <code>sleep</code> command. Almost all programming languages have this feature, and is used in many use-cases.</p>

<h3>time.sleep() Arguments</h3>
<ul>
<li><strong>secs</strong> - The number of seconds the Python program should pause execution. This argument should be either an <code>int</code> or a <code>float</code>.</li>
</ul>

<h2>The Accuracy of time.sleep()</h2>
<p>The <code>time.sleep()</code> function uses the underlying operating system's <code>sleep()</code> function. Ultimately there are limitations of this function. For example on a standard Windows installation, the smallest interval you may <em>sleep</em> is 10 - 13 milliseconds. The Linux kernels tend to have a higher tick rate, where the intervals are generally closer to 1 millisecond. Note that in Linux, you can install the <code>RT_PREEMPT</code> patch set, which allows you to have a semi-realtime kernel. Using a real-time kernel will further increase the accuracy of the <code>time.sleep()</code> function. Generally however, unless you want to <em>sleep</em> for a very small period, you can generally ignore this information.</p>
