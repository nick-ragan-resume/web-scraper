<div class="s-prose js-post-body" itemprop="text">
<p>TL;DR set the env. vars. mentioned in <code>tcl-tk</code>'s caveats <strong>and</strong> <a href="https://github.com/pyenv/pyenv/issues/1375#issuecomment-533182043" rel="noreferrer">this GitHub comment</a> when installing new Pythons via pyenv to get <code>tkinter</code>.</p>
<p>First, ensure you have the latest <code>tcl-tk</code> via homebrew and then pay attention to its caveats:</p>
<pre><code>※ brew install tcl-tk
※ brew info tcl-tk
tcl-tk: stable 8.6.10 (bottled) [keg-only]
...
==&gt; Caveats
tcl-tk is keg-only, which means it was not symlinked into /usr/local,
because tk installs some X11 headers and macOS provides an (older) Tcl/Tk.

If you need to have tcl-tk first in your PATH run:
  echo 'export PATH="/usr/local/opt/tcl-tk/bin:$PATH"' &gt;&gt; ~/.zshrc

For compilers to find tcl-tk you may need to set:
  export LDFLAGS="-L/usr/local/opt/tcl-tk/lib"
  export CPPFLAGS="-I/usr/local/opt/tcl-tk/include"

For pkg-config to find tcl-tk you may need to set:
  export PKG_CONFIG_PATH="/usr/local/opt/tcl-tk/lib/pkgconfig"
...
</code></pre>
<p>You'll also need to know about pyenv's <code>PYTHON_CONFIGURE_OPTS</code>, <code>--with-tcltk-includes</code>, and <code>--with-tcltk-libs</code>, e.g. from <a href="https://github.com/pyenv/pyenv/issues/1375#issuecomment-533182043" rel="noreferrer">this comment</a>.</p>
<p>Next, reinstall Python with the environment variables active:</p>
<pre><code>※ pyenv uninstall 3.8.1
※ env \
  PATH="$(brew --prefix tcl-tk)/bin:$PATH" \
  LDFLAGS="-L$(brew --prefix tcl-tk)/lib" \
  CPPFLAGS="-I$(brew --prefix tcl-tk)/include" \
  PKG_CONFIG_PATH="$(brew --prefix tcl-tk)/lib/pkgconfig" \
  CFLAGS="-I$(brew --prefix tcl-tk)/include" \
  PYTHON_CONFIGURE_OPTS="--with-tcltk-includes='-I$(brew --prefix tcl-tk)/include' --with-tcltk-libs='-L$(brew --prefix tcl-tk)/lib -ltcl8.6 -ltk8.6'" \
  pyenv install 3.8.1
</code></pre>
<p>It should work now:</p>
<pre><code>※ pyenv global 3.8.1
※ python
Python 3.8.1 (default, Feb 29 2020, 11:56:10)
[Clang 11.0.0 (clang-1100.0.33.17)] on darwin
Type "help", "copyright", "credits" or "license" for more information.
&gt;&gt;&gt; import tkinter
&gt;&gt;&gt; tkinter.TclVersion, tkinter.TkVersion
(8.6, 8.6)
&gt;&gt;&gt; tkinter._test()
# You should get a GUI
</code></pre>
<p>If you get the following error, you might be missing the <code>PYTHON_CONFIGURE_OPTS</code> env. var. above.</p>
<pre><code>DEPRECATION WARNING: The system version of Tk is deprecated and may be removed in a future release. Please don't rely on it. Set TK_SILENCE_DEPRECATION=1 to suppress this warning.
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
  File "/Users/factor/.pyenv/versions/3.8.1/lib/python3.8/tkinter/__init__.py", line 4552, in _test
    root = Tk()
  File "/Users/factor/.pyenv/versions/3.8.1/lib/python3.8/tkinter/__init__.py", line 2263, in __init__
    self._loadtk()
  File "/Users/factor/.pyenv/versions/3.8.1/lib/python3.8/tkinter/__init__.py", line 2279, in _loadtk
    raise RuntimeError("tk.h version (%s) doesn't match libtk.a version (%s)"
RuntimeError: tk.h version (8.6) doesn't match libtk.a version (8.5)
</code></pre>

<p>Here is <strong>step by step guide</strong> to make <code>tkinter</code> (and IDLE) work if you use <code>pyenv</code> for Python environments management on macOS:</p>
<ol>
<li>install <code>tcl-tk</code> with Homebrew. In shell run <code>brew install tcl-tk</code></li>
<li>in shell run <code>echo 'export PATH="/usr/local/opt/tcl-tk/bin:$PATH"' &gt;&gt; ~/.zshrc</code></li>
<li>reload shell by quitting <code>Terminal</code> app or run <code>source ~/.zshrc</code></li>
<li>after reloaded check that <code>tck-tk</code> is in <code>$PATH</code>. Run <code>echo $PATH | grep --color=auto tcl-tk</code>. As the result you should see your $PATH contents with <code>tcl-tk</code> highlighted</li>
<li>now we run three commands from Homebrew's output from step #1

<ol>
<li>in shell run <code>export LDFLAGS="-L/usr/local/opt/tcl-tk/lib"</code></li>
<li>in shell run <code>export CPPFLAGS="-I/usr/local/opt/tcl-tk/include"</code></li>
<li>in shell run <code>export PKG_CONFIG_PATH="/usr/local/opt/tcl-tk/lib/pkgconfig"</code></li>
</ol></li>
<li>if you have your Python version already installed with <code>pyenv</code> then uninstall it with <code>pyenv uninstall &lt;your python version&gt;</code>. E.g. <code>pyenv uninstall 3.8.2</code></li>
<li>set environment variable that will be used by <code>python-build</code>. In shell run <code>PYTHON_CONFIGURE_OPTS="--with-tcltk-includes='-I/usr/local/opt/tcl-tk/include' --with-tcltk-libs='-L/usr/local/opt/tcl-tk/lib -ltcl8.6 -ltk8.6'"</code>
<em>Note: in future use <code>tck-tk</code> version that actually installed with Homebrew. At the moment of posting <code>8.6</code> was the actual</em></li>
<li>finally install Python with <code>pyenv</code> with <code>pyenv install &lt;version&gt;</code>. E.g. <code>pyenv install 3.8.2</code></li>
</ol>
<p><strong>Test</strong></p>
<ol>
<li>in shell run <code>pyenv global &lt;verion that you've just installed&gt;</code></li>
<li>now check IDLE. In shell run <code>idle</code>. You should see IDLE window without any warnings and "text printed in red".</li>
</ol>
<p><a href="https://i.stack.imgur.com/tysBK.png" rel="noreferrer"><img alt="IDLE window run from Terminal. No warnings" src="https://i.stack.imgur.com/tysBK.png"/></a></p>
<ol start="3">
<li>now check <code>tkinter</code>. In shell run <code>python -m tkinter -c "tkinter._test()"</code>. You should see test window like on the image:</li>
</ol>
<p><a href="https://i.stack.imgur.com/hDBOH.png" rel="noreferrer"><img alt="tkinter test window" src="https://i.stack.imgur.com/hDBOH.png"/></a></p>
<p>That's it!</p>
<p><strong>My environment:</strong></p>
<p>check this is something went wrong executing steps above:</p>
<ol>
<li>macOS Catalina</li>
<li><code>zsh</code> (included in macOS Catalina) = <strong>"shell" above</strong></li>
<li>Homebrew (installed with instructions from Homebrew official website)</li>
<li><code>pyenv</code> (installed with Homebrew and PATH updated according to <code>pyenv</code> official readme from GitHub)</li>
<li>Python <code>3.8.x</code> - <code>3.9.x</code> (installed with <code>pyenv install &lt;version&gt;</code> command)</li>
</ol>

<p>I had the same issue when I tried to install <code>tkinter</code> through pyenv. I was able to fix it using the following in case someone has the same problem and still want to stick with <code>pyenv</code>.</p>
<p><strong>The Fix</strong></p>
<ol>
<li><p>I followed @nickolay instructions to install <code>tkinter</code> and set the path the proper way.</p>
</li>
<li><p>Then, I installed anaconda3-2020.07 using the <code>pyenv install anaconda3-2020.07</code> command.</p>
</li>
<li><p>Because I am using pyenv python 3.8.6 <strong>globally</strong>: I navigated to the folder I want to use tkinter and used the <code>anaconda3-2020.07</code> <strong>locally</strong> by utilizing the command <code>pyenv local anaconda3-2020.07</code>to use this version of pyenv in that specific folder. It ran without errors!</p>
</li>
</ol>
<p><strong>Note:</strong> I am using the following script in the <code>.bash_profile</code> to trigger the virtualenv automatically when cd the desired directory</p>
<pre class="lang-sh prettyprint-override"><code># manage python version using pyenv
if command -v pyenv 1&gt;/dev/null 2&gt;&amp;1; then
  eval "$(pyenv init -)"
fi

# add pyenv virtualenv
eval "$(pyenv virtualenv-init -)"
</code></pre>
