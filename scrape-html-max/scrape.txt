        self.person_name = person_name
        self.health = health
        Weapon.ranged_weapon = ranged_weapon

    def display_person_info(self):
        print('Name: %s' %self.person_name)
        print('Ranged Weapon :%s' %Weapon.display_weapon_name)

def ranged_attack(self, ranged_weapon, target):
    target.health -=ranged_weapon.weapon_damage
    print("Weapon: %s" %ranged_weapon.weapon_name)
    print(target.person_name + "'s Health: "+str(target.health))

pistol = Weapon("Pistol", 40)
bob = Person("Bob", 100, pistol)

bob.display_person_info()
</code></pre>
<p>Running this gives me:</p>
<pre><code>Name: Bob
Ranged Weapon :&lt;function Weapon.display_weapon_name at 0x02E23030&gt;
</code></pre>
<p>Running:</p>
<pre><code>bob.ranged_attack(pistol, bob)
</code></pre>
<p>Gives:</p>
<pre><code>Weapon: Pistol
Bob's Health: 60
</code></pre>
<p>My questions are, am I passing the Weapon object correctly to the Person class? It seems weird writing Weapon.ranged_weapon in _init__ rather than self.ranged_weapon.</p>
<p>How can I get the display_weapon_info to show the string 'Weapon Name: Pistol', rather than the reference? It seems to work when I call it in ranged_attack, but not in the display info.</p>
<p>Really appreciate any help I can get with this. Apologies if a similar question has been asked before, but I couldn't find anything I could relate to my issue.</p>

<p><code>Person</code> doesn't actually need to reference the <code>Weapon</code> class directly; it just needs to save a reference to whatever is passed as the <code>ranged_weapon</code> argument and know what it can do with that object. The code implicitly assumes that <code>ranged_weapon</code> is an instance of <code>Weapon</code>, but will work with any object that is suitably similar to an instant of <code>Weapon</code>.</p>
<pre><code>class Person:

    def __init__(self, person_name, health, ranged_weapon):
        self.person_name = person_name
        self.health = health
        self.weapon = ranged_weapon

    def display_person_info(self):
        print('Name: %s' %self.person_name)
        # display_weapon_name already calls print; but
        # you probably don't need this method at all.
        self.weapon.display_weapon_name()
        # Instead, do this (actually, you already do this
        # in ranged_attack())
        # print('Weapon: %s' % self.weapon.weapon_name)


    def ranged_attack(self, ranged_weapon, target):
        target.health -= self.weapon.weapon_damage
        print("Weapon: %s" % self.weapon.weapon_name)
        print(target.person_name + "'s Health: "+str(target.health))
</code></pre>

<div class="s-prose js-post-body" itemprop="text">
<pre><code>def display_person_info(self):
        print('Name: %s' %self.person_name)
        print('Ranged Weapon :%s' %Weapon.display_weapon_name)
</code></pre>
<p>Looking at this function, the compiler sees the following:</p>
<p>Line 1: A function named <code>display_person_info</code> with the parameter <code>self</code>.<br/>
Line 2: Print "Name: " and then print the name of <code>self</code><br/>
Line 3: Print "Ranged Weapon: " and then <code>Weapon.display_weapon_name</code>.</p>
<p>In line 3, the compiler, rather than printing the weapon name, it is printing the function <code>display_weapon_name</code> itself! What you need to do instead is replace Line 3 with this:</p>
<pre><code>print('Ranged Weapon :%s' %self.weapon.display_weapon_name())
</code></pre>
<p>That way, the value returned by the function is printed, rather than the function's pointer.</p>
